D Programming Language for HPC

The question I want to investigate is if the D Programming Language will be feasible for high-performance computing.

  • Can the compiler produce 64-bit executables? Yes
  • Can executables run on capacity clusters? Yes, tested on glyph and its nodes
  • Can executables run on high-end supercomputers?
  • Can the linker link with pthreads?
  • Can the linker link with BLAS and LAPACK? Yes, tested with dsyev_ entry
  • Can the compiler call MPI? Can the linker link with MPI?
  • Is there a slowdown?
